# 棋子go

Photoshop UXP 本地插件，用于读取腾讯文档导出的 `.xlsx`，定位表格中的资源代码与嵌入图片，并生成归档 PSD。

当前完成状态：

- Phase 0 技术尖峰已在 Windows / Photoshop 2024 真机通过；
- Phase 1 XLSX 只读解析器已完成并通过两个真实样本验证；
- Phase 2 映射、校验、分页选择和解析 Manifest 已完成代码与自动化验证，等待最终 Photoshop UI 人工复核；
- Phase 3 的核心生成、重开、精确边界、取消和空白智能对象真机验收已通过；
- Phase 4 产品化进行中，已完成界面精简、诊断包导出、大文件提示和最近文件恢复；跨平台烟测与正式打包仍待完成。

## 已完成能力

- TypeScript strict、React 18、Webpack 5 的 UXP panel；
- Photoshop 2023 24.2+ 兼容 Manifest v5（`minVersion: 24.2.0`、`apiVersion: 2`）；
- UXP 本地 `.xlsx` 二进制选择与读取；
- 直接读取 ZIP/XML，不依赖 `styles.xml` 或完整 Excel 对象模型；
- 解析 workbook、relationships、shared strings、sheet cells；
- 解析 drawing relationships、`oneCellAnchor`、`twoCellAnchor`；
- 支持腾讯 `xl/drawings/media` 与标准 `xl/media` 图片路径；
- 按工作表读取单元格与图片锚点，不预解压全部图片；
- 资源映射、缺失/重复/多图片候选校验与 A 列分组多选；
- 连续同名 A 列分段自动合并，带资源的空白分段可作为前组续段；
- 棋子链分组默认展开并可独立折叠，多排候选支持按图片排位批量选择；
- 缩略图按面板可视区域延迟加载，实时数据 URI 与 LRU 缓存均限制为 32 项；
- 独立导出包含来源地址、候选图片和校验问题的解析 Manifest；
- 导出不含 XLSX 与图片二进制的诊断 ZIP，包含摘要、解析 Manifest 和结构化日志；
- 超过 250 MB 的工作簿显示大文件模式提示；
- 通过 UXP persistent token 记住最近选择的 XLSX；令牌失效时清除记录并回退到重新选择；
- 工作簿导入后自动选择并解析第一个工作表；切换工作表时自动读取棋子链，分组默认不选；
- 范围列表工具栏显示已选数量，并提供“全选 / 清空全选”两态按钮；
- 无关联图片只提示，不阻断 PSD 生成；对应画板仍包含可编辑空白智能对象，但不放参考图；
- 运行日志和诊断工具默认收起，日志使用宿主灰色背景的只读文本区，可用鼠标拖选并通过 `Ctrl+C` 复制；更新只由每 10 分钟一次的后台定时器或用户点击“检查更新 / 重新检查”触发，插件启动、窗口重新聚焦和展开诊断页不会请求 GitHub。发现更高版本时，“运行与诊断”标题旁显示“（有新版本）”，展开后可直接启动内部更新；
- 主面板只显示生成画板按钮；点击后在 PSD 保存弹窗中填写名称和选择位置，只保存 PSD，后续分卷自动使用同一前缀；
- 生成前可设置正方形空白智能对象的边长和画板间距，默认分别为 1024 px 与 50 px；插件会记住上次输入；置入画板后的空白智能对象仍保持 148×148 px；
- “AI初稿”提供 Flux、GPT Image 2 与 G+F 三种隔离工作流。G+F 先用 GPT Image 2 生成完整物件链并保存 raw sheet，随后按阅读顺序裁成单件图；裁切批次及截断前检测数量必须与物品数完全一致，否则在任何 Holopix 付费分支前终止。每张裁切图再分别经 `HolopixUploadReference` 接入独立的原始多模型 `HolopixGenerate.reference`，固定以 `0.4` 参考权重串行细化，最后按原顺序合批、抠图并以稳定 `assetCode` 回填候选矩阵；
- Phase 0 的新文档、148×148 画板、嵌入式智能对象、contain 定位与 PSD 保存链路；
- 每个棋子画板使用默认 RGB(199, 212, 226) 的原生画板背景，生成后可从“当前 PSD”统一拾色改色或显示/隐藏；
- Photoshop 中打开任意文档时，都会显示默认收起的一级折叠栏“AI编辑”和“AI勾线”，没有文档时两栏隐藏。AI编辑会把当前图层提交到本地 V2/V3 工作流并保留就绪结果；AI勾线直接读取当前活动图层并提交 Centerline Forge 工作流。两者最终写入时都重新验证来源文档和来源图层；AI勾线可在画板、标准网格和普通文档中按来源位置直接插入，AI编辑仍只在受支持画布中自动写入；
- AI勾线顶部始终提供“生成描边”；“插入就绪描边”只绑定当前插件会话中生成该 Path JSON 的精确 Photoshop 文档 ID 与图层 ID，并按保存的图层 ID 直接读取来源图层，因此生成出的 `Autoline` 或其他图层处于选中状态时也可直接插入，无需手动切回来源图层。ComfyUI 已完成但任务期间活动文档或图层发生变化时，插件不再把生成本身判为失败，而是保留结果并显示“描边已就绪，等待插入”。切换到其他文档或删除来源图层时按钮隐藏，切回来源文档后恢复；若来源图层恰好在点击后失效则明确报错。插件重载后不会仅凭 ComfyUI 全局最近输出推断来源。上下文检测使用 Photoshop 事件通知，只遍历图层 ID，不读取像素、不轮询，也不增加生成任务。描边宽度常驻显示，精细度、折角敏感度和平滑度收入“高级选项”。高级标题与展开箭头作为一组靠左显示，展开内容与标题共用一个无内部分割线的外框；四个滑杆右侧均提供数字输入框，生成的工作路径与 Shape 图层统一命名为 `Autoline`。内部工作路径在 Shape 成功后自动清理，仅在转换失败时保留用于恢复；任务进度卡右下角提供取消入口，详细状态统一进入“运行与诊断”；
- AI勾线的完整 ComfyUI 任务等待上限为 360 秒，超过 200 秒的 Holopix 工作流仍会继续轮询；到达上限时还会执行一次最终历史检查，避免结果刚写完却被误报为超时；
- 每次点击 AI勾线“生成描边”都会使用独立的上传子目录和新的 Holopix `request_nonce`，即使当前图层像素与上一次完全相同，也会重新提交并执行完整图像生成链。“插入就绪描边”在高级参数未变时不会新增 ComfyUI 任务；若精细度、折角敏感度或平滑度发生变化，则复用上次生成的中间图像，仅执行矢量化与保存节点，不重新上传图层或运行 Holopix；
- Shape 创建成功后，AI勾线按本次唯一名称与路径 ID 重新定位并清理中间工作路径；若 Photoshop 已在转换时自动消耗路径则直接视为完成，Photoshop 25.4 的 PathItem 未提供 `remove()` 时改用 Action 删除，不影响已经创建的 Shape；
- 解析器、路径、A 列分组、A1 工具、布局与输出工具的自动化测试。

## 标准网格画布

棋子go 只支持一套非画板标准模板：画布 `1780 × 1188 px`，`12 × 8` 共 96 格；每格实体区域 `144 × 144 px`，水平/垂直间距均为 `4 px`，四周外边距均为 `4 px`。没有 Photoshop 画板且尺寸恰好匹配时，面板会提示“尚未初始化”；点击“初始化标准网格”只创建隐藏配置组与文本层，不创建视觉网格，也不移动、缩放或删除设计图层。保存、关闭并重开 PSD 后仍会从该配置层识别。

展开“生成 PSD”后，“生成网格画布”会以与“选择 XLSX”相同的蓝色主按钮显示在导入表格区域下方，不再单独占用一级面板。该按钮不需要先导入 Excel 或打开其他文档；选择保存位置后，插件会创建 `1780 × 1188 px` 的非画板 RGB PSD，把随插件发布的标准 `12 × 8` 背景图作为 `__BACKGROUND__` 智能对象精确铺满画布，并同时写入隐藏网格数据。保存后的文档可立即被识别为标准网格并用于 AI初稿整排/单个回填。

布局扫描按需发生：进入一份已初始化网格文档后的首次使用、准备放置 AI 结果、需要重新规划空位、内存中没有该 documentId 的有效缓存，或用户点击“刷新布局”时。插件不会因每次移动、粘贴或变换图层而扫描，也不会把动态占用写入 PSD。写入前还会仅复检计划使用的目标格；如果设计师刚放入了内容，本次写入会取消并重新给出建议位置。

占用扫描只读取顶层图层的 `boundsNoEffects`，不读取像素、不检查 Alpha、不递归组内图层。相交面积至少达到图层边界面积与单格实体面积中较小者的 20% 才算占用，因此棋子轻微越过格线不会误占相邻格，真正明显跨格的内容仍会占用多格。隐藏的设计图层仍占位；边界覆盖完整 `1780 × 1188 px` 画布的网格底板或背景层会按几何特征忽略，不依赖图层名称。以下顶层名称也会被忽略：

- `棋子go｜标准网格画布数据`
- `__GRID__`
- `__BACKGROUND__`
- `__NOTES__`
- `__REFERENCES__`
- `__ARCHIVE__`

候选生成区在界面中命名为“AI初稿”。画板 PSD 中的候选点击与回填方式保持不变。标准网格下始终显示 `AI初稿` 入口；如果插件在网格标签页上重载，会自动扫描其他已打开的 Photoshop 文档并恢复来源画板链。切换到标准网格 PSD 后，只要提供候选链的来源画板 PSD 仍保持打开，Flux、GPT Image 2 与 G+F 的继续生成、候选数量调整和 Flux 单格重生成仍可使用；提交前会按来源 documentId 验证原 PSD 和节点范围，但不要求原 PSD 是当前活动文档。标准网格中，无论使用哪一种 AI初稿工作流，点击任意候选都会选择该候选列，并提供“插入整排”和“插入当前”两个入口：前者写入该候选列中已有图片的全部链位，后者只写入本次点击的棋子。物品链最多 12 个位置，首次写入只使用 12 格全部无内容的第一条完整空行，并新建一个以链名称开头的顶层 `AI初稿` 图层组，把该链及绑定行写入组名；整排或单个插入的智能对象都放在该组内。链中第 N 个棋子固定对应该行第 N 格，缺失位置保持为空，后面的棋子不会前移补位。后续再选择同一条链时，按钮变为“更新整排”和“更新当前”，继续使用原组：已有链位直接替换原嵌入式智能对象内容，尚未写入的固定链位可继续补齐，永远不会重复建组或为同一条链再开第二排。写入前会排除本链组后复检目标格，并单独检查组内其他内容；若存在不属于目标链位的手工内容则停止覆盖。图片保持比例、完整 contain、居中，默认不放大低分辨率图片；建组、迁移旧版散层和同一批写入均由单个可回滚历史事务保护。

`AI初稿` 的可见性只取决于当前活动文档：画板文档和已初始化标准网格显示；普通非画板且非标准网格的文档隐藏。切到普通文档时不会清空已打开来源 PSD 的候选和链状态，之后切回标准网格仍可恢复。

AI编辑、AI勾线与 AI细化不是新棋子，不寻找空格，也不新增网格身份。它们会验证生成时保存的来源 documentId/layerId，把结果放在来源图层上方（或沿用 AI编辑的“文档最上方”选项），使用来源边界完成既有对齐；其中 AI细化在标准网格中还会按每个来源图层重叠面积最大的主格限制最终边界，优先保留来源尺寸与中心，必要时平移或缩放，确保不越出 `144 × 144 px` 实体格。AI编辑和 AI细化默认把下载结果立即包装为 Photoshop 原生嵌入式 PSB 智能对象，再执行定位与缩放；高级选项允许明确改为栅格图层。原图层保留，网格占用缓存不增加新格。普通画布允许 AI勾线按来源位置插入；AI编辑和 AI细化的候选/就绪结果仍会保留，供用户切回受支持文档处理。

整组 AI细化的输出组仍命名为“`原组名 细化`”，但其子图层统一追加“细化”后缀，不复用 `AI初稿 …｜链标记-位置` 绑定名。标准网格读取链绑定时也会忽略已有“AI初稿绑定组名 + 细化”派生组中的旧同名子图层，因此旧 PSD 无需手动改名，后续更新整排或单个链位不会再误报“位于绑定组外”。

标准网格 Photoshop 真机验收步骤见 [`docs/STANDARD_GRID_MANUAL_TESTS.md`](docs/STANDARD_GRID_MANUAL_TESTS.md)。

## Windows 分发安装

`release` 分支根目录包含纯 ASCII、全英文 `install.cmd`。用户可以只下载这一份 CMD：双击后安装器会先自更新，再自动定位或通过 winget 安装 Git，并通过 GitHub API 比较 `release` 版本；有更新时优先拉取 Git，Git 通道不可用时自动改用 GitHub Codeload 发布压缩包。首次安装时选择一次包含 `Photoshop.exe` 的 Photoshop 根目录。

安装后的运行包同时包含经发布清单校验的 `ChessGoInstaller.cmd`。插件检测到新版本后会把该安装器复制到 UXP 临时目录，并通过用户授权的 `shell.openPath` 打开临时 `.vbs` 无窗口启动器；底层安装器在后台运行，不显示命令行窗口。安装器把管理员授权、环境检查、发布包下载与校验、文件写入和最终结果作为 UTF-8 JSONL 状态写到同一临时目录，面板持续读取并追加到“运行与诊断”日志。内部模式沿用当前 UXP 注册位置，不再选择 Photoshop 路径。管理员授权、安装器自更新、固定 release 提交下载、逐文件 SHA-256 校验、原子替换与旧版本备份仍由同一安装器负责，成功后弹窗要求重启 Photoshop。

内部更新能力从 `0.8.8` 开始提供，因此 `0.8.7` 及更早版本仍需最后手动运行一次外部 `install.cmd` 升级到 `0.8.8`；之后的新版本才可直接从 UXP 面板更新。

发布脚本会递归收集 `dist` 中的全部文件并生成带文件大小和 SHA-256 的 `release-manifest.json`；发布分支用 `.gitattributes` 保持这些文件的原始字节不被 Git 换行转换。安装器按清单自动创建子目录、复制并逐项校验，因此后续新增运行文件不再需要修改安装器清单，Git 克隆与 Codeload ZIP 也使用同一套哈希。安装器会请求管理员权限、开启 UXP developer mode，把完整运行包原子写入 Adobe UXP `Plugins\External`，并更新 `PS.json` 注册。不依赖 Creative Cloud、UXP Developer Tool、CCX 或 UPIA。若 Photoshop 正在运行，安装后需重启一次 Photoshop。

## 安装依赖与构建

需要 Node.js 20+ 和 pnpm：

```text
pnpm install
pnpm verify
```

`pnpm verify` 会依次执行 TypeScript 类型检查、单元测试和生产构建。可加载产物位于 `dist/`。

## 在 UXP Developer Tool 中加载

1. 打开 Photoshop 2023 24.2 或更高版本，并启用开发者模式。
2. 打开 Adobe UXP Developer Tool。
3. 点击 **Add Plugin**。
4. 选择 `D:\Scripts\UXP\PsdArchive\dist\manifest.json`。
5. 点击插件右侧菜单并选择 **Load**。
6. 在 Photoshop 的“增效工具 / 插件”菜单中打开“棋子go”。

修改源码后先执行 `pnpm build`，再在 UXP Developer Tool 中 Reload。

## XLSX 解析诊断

输出工作簿内所有工作表的解析摘要：

```text
pnpm inspect:xlsx "D:\path\sample.xlsx"
```

只检查一个工作表，并输出指定 assetCode 的单元格、图片 entry 与锚点：

```text
pnpm inspect:xlsx "D:\path\sample.xlsx" --sheet "越南第三十九至四十一章" --codes "ds_vietnam1,ds_vietnam3,ds_vietnam52"
```

真实样本的已记录结果见 `docs/PHASE1_PARSER_SUMMARIES.md`。
