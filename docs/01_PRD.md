# 棋子go — 产品需求文档（PRD）

- 文档版本：1.0
- 产品形态：Photoshop UXP 面板插件
- 运行方式：纯本地、离线可用
- 输入：从腾讯文档下载的 `.xlsx` 副本
- 输出：归档 `.psd` / 必要时 `.psb`

---

## 1. 背景

当前棋子归档过程包含大量确定性人工操作：从排期表中确认资源代码和图片、逐个创建画板、输入画板名、置入图标、调整位置、检查漏图与错名，再保存 PSD。现有排期文件体积很大，包含大量嵌入图片；现有归档 PSD 则表现出稳定、可程序化的结构。

本项目不解决“AI 如何画图”，只解决：

> 把一个固定版本的腾讯文档 Excel 副本，可靠地转换为结构正确、命名正确、可编辑的归档 PSD。

选择本地副本而不是链接直读，是为了省去 OAuth、后台服务、网络失败、实时版本漂移和腾讯文档接口维护，并让一次 PSD 归档天然对应一个固定数据快照。

## 2. 样本基线

### 2.1 Excel 样本

当前样本：

- `M图标月度安排1.xlsx`：约 77 MB，23 个工作表，约 2204 个嵌入图片；
- `M图标月度安排2.xlsx`：约 196 MB，12 个工作表，约 2546 个嵌入图片。

腾讯文档导出的文件存在非标准实现细节：

- 图片位于 `xl/drawings/media/`，而不一定是常见的 `xl/media/`；
- 图片通过工作表关系、drawing XML、drawing relationship 三层关系定位；
- 图片常用 `oneCellAnchor` 锚定到单元格，也要兼容 `twoCellAnchor`；
- `styles.xml` 会让部分通用解析器报错，因此 MVP 不读取样式；
- 同一资源代码下可能存在两个图片候选，不能假设“每个项目只有一张图片”。

### 2.2 归档 PSD 样本 `02.psd`

实测结构：

- 文档：2380 × 1140 px，RGB，8 bit，300 ppi；
- 43 个画板；
- 每个画板：148 × 148 px；
- 横向、纵向间距：100 px，即网格步长 248 px；
- 每个画板恰好包含一个嵌入式智能对象；
- 画板名使用资源代码，例如 `ds_vietnam3`；
- 可见内容水平中心中位数为 74 px，垂直中心中位数为 78 px，即相对几何中心下移约 4 px；
- 可见内容最大约 145 × 133 px；
- 样本不是把 43 项连续紧凑排列，而是保留了两个源分组：第一组 14 项占两行，第二组从新行开始并占三行。

因此，默认布局必须支持“保留源分组换行”，而不是只做连续 row-major 排列。

## 3. 产品目标

### 3.1 核心目标

用户在 Photoshop 内完成以下闭环：

1. 导入 `.xlsx` 副本；
2. 选择工作表；
3. 按最左侧 A 列分组选择范围；
4. 查看解析出的棋子需求；
5. 处理缺失、重复和多图片候选；
6. 选择要归档的项目和排列方式；
7. 使用模板生成 PSD；
8. 保存 PSD。

### 3.2 成功指标

- 选择文件到看到工作表列表：目标 5 秒内；大文件允许 15 秒并显示进度；
- 选择单个工作表后完成文本和图片索引：目标 10 秒内；大工作表允许 30 秒；
- 43 项基准数据生成 PSD：目标 3 分钟内；
- 100 项生成成功率：≥ 99%，异常项目必须有明确报告；
- 生成后画板名、数量和顺序准确率：100%；
- 不允许因某一个异常项目导致已选择项目被静默漏掉。

以上是工程目标，不是承诺值；Phase 0 后根据实际 Photoshop 环境校准。

## 4. 用户角色

### 4.1 主要用户：美术/棋子制作人员

需要快速把已完成图标归档到 PSD，不希望理解 Excel OOXML 或 Photoshop 脚本。

### 4.2 次要用户：美术统筹

需要确认本批次包含哪些资源、有哪些异常、最终 PSD 对应哪个 Excel 副本和工作表。

## 5. 用户流程

### 5.1 导入

用户点击“导入 Excel 副本”，通过系统文件选择器选择 `.xlsx`。

插件显示：

- 文件名；
- 文件大小；
- 最后修改时间（若 UXP 元数据可得）；
- 工作表列表；
- 导入状态和错误。

不支持 `.xls`、在线链接和加密工作簿。

### 5.2 选择工作表和 A 列分组

用户选择一个工作表，插件解析最左侧 A 列的分组标签，然后允许多选一个或多个分组：

- 优先使用 A 列 `mergeCells` 的起止行作为分组边界；
- 没有合并信息时，从当前 A 列标签延伸到下一个标签之前；
- 一个分组右侧的一行或多行内容全部纳入，不要求用户填写 A1 地址；
- 获取完成后展开范围，默认不选择任何分组；
- 使用一个“全选 / 清空全选”两态按钮，并允许任意多选分组；
- 空白但有内容的合并分组显示为“未命名分组（第 X–Y 行）”。

项目是否属于分组以 **资源代码单元格的行号** 为准。名称、numericId 和图片锚点仍可位于 code 行上下的关联位置。

### 5.3 解析与预览

插件以资源代码单元格为锚点识别项目。每项展示：

- 是否选中；
- `assetCode`；
- `numericId`；
- 中文名称；
- 来源单元格；
- 来源分组；
- 图片缩略图；
- 校验状态。

支持筛选：

- 全部；
- 可生成；
- 缺图片；
- 多图片候选；
- 缺 numericId；
- 重复 assetCode；
- 按前缀 `ds/c/g/m/box/other`。

支持排序：

- 源表顺序（默认）；
- assetCode；
- numericId；
- 名称。

MVP 不要求拖拽排序。

### 5.4 多图片候选

若一个项目映射出多张图片，插件必须：

- 将上排图片明确标记为“参考图”；
- 将下排图片明确标记为“项目图”；
- 默认选择下排项目图，同时展示全部候选缩略图和来源锚点；
- 允许用户随时切换选择，多图片本身不作为错误。

不能静默选择上排参考图。

### 5.5 模板设置

默认模板“归档 148”参数：

- 画板：148 × 148 px；
- 列数：10；
- 横向间距：100 px；
- 纵向间距：100 px；
- 分辨率：300 ppi；
- RGB / 8 bit；
- 每个棋子画板底层包含独立的 `底板颜色` 填充层，默认 RGB(199, 212, 226)；
- 生成后可在“当前 PSD”中通过 Photoshop 拾色器统一修改全部底板颜色，并统一显示或隐藏底板；
- 内容可见边界：最大 146 × 134 px；
- 内容中心：画板坐标 (74, 78)；
- 缩放策略：等比 contain，只缩小，不放大；
- 置入方式：嵌入式智能对象；
- 画板名：assetCode；
- 子图层名：assetCode；
- 保留源分组换行：开启。

用户可调整画板尺寸、列数、间距、内容框、垂直偏移、是否允许放大、是否紧凑排列。用户设置可保存为本地模板。

### 5.6 生成前检查

点击“生成”前显示摘要：

- 已选项目数；
- 预计画板行数；
- 预计文档尺寸；
- 阻断错误数；
- 警告数；
- 输出文件数。

阻断错误为 0 才允许生成。

### 5.7 保存

点击生成后打开一次 PSD 保存弹窗，用户在弹窗中同时选择保存位置和填写首卷名称。插件只保存 PSD；存在后续分卷时自动在同一目录继续保存，不重复弹窗。

默认输出：

```text
{工作簿名}_{工作表名}.psd
```

单卷时，用户选择的 PSD 名称即为最终文件名。多卷时，弹窗默认建议 `_01.psd`；后续卷自动使用 `_02.psd`、`_03.psd` 等名称。

若超过模板设置的单文件上限，则生成 `_02`、`_03`。

## 6. 功能需求

### FR-01 文件导入

- 通过 UXP 文件选择器选择 `.xlsx`；
- 文件读取失败时给出可操作错误；
- 记住最近一次文件引用，失效时重新请求；
- 不需要网络权限。

### FR-02 工作表索引

- 解析 `xl/workbook.xml` 与 workbook relationships；
- 保留工作表原始顺序和 Unicode 名称；
- 不读取样式和公式计算结果；
- 隐藏工作表可显示但要标记。

### FR-03 单元格读取

至少支持：

- shared string；
- inline string；
- 数值；
- 布尔；
- 空单元格；
- rich text shared string 的文本合并。

所有 ID 按字符串处理，禁止转成会丢失前导零的 number。

### FR-04 图片索引

- 解析 sheet drawing relationship；
- 支持 `oneCellAnchor` 和 `twoCellAnchor`；
- 解析图片 relationship；
- 正确归一化 `../` 相对路径；
- 兼容 `xl/drawings/media/` 和 `xl/media/`；
- 只在需要时解压图片二进制。

### FR-05 项目识别

默认识别逻辑：

1. 在选定 A 列分组覆盖的行段内寻找符合 assetCode 规则的单元格；
2. 同列向上寻找名称和 numericId；
3. 同列向下寻找图片锚点；
4. 同一资源代码行形成 `sourceGroup`；
5. 按代码单元格的行、列顺序排序。

默认偏移：

- 名称：codeRow - 1；
- numericId：codeRow - 2；
- 图片候选：codeRow + 1、codeRow + 2。

同时提供“向上最多 4 行、向下最多 3 行”的容错扫描。若精确偏移和扫描结果冲突，标记为需要确认。

### FR-06 资源代码规则

默认合法规则：

```regex
^[A-Za-z][A-Za-z0-9_]*$
```

可选前缀白名单：`ds_`、`c_`、`g_`、`m_`、`box_`、`dish_`。非白名单但符合通用规则的项目归为 `other`，不自动报错。

### FR-07 校验

阻断错误：

- assetCode 为空；
- assetCode 非法；
- assetCode 在当前任务中重复；
- 未找到图片；
- 图片存在但没有任何有效选择；
- 图片文件损坏或 Photoshop 无法置入。

警告：

- numericId 为空；
- 名称为空；
- numericId 重复；
- 非标准前缀；
- 图片像素尺寸过小；
- 关联字段位于所选分组边界外。

### FR-08 项目选择

- 全选、全不选；
- 按过滤结果全选；
- 单项选择；
- 阻断错误项默认不可选；
- 用户修复图片候选后可重新选择。

### FR-09 分组布局

默认保留源分组：

- 同一 codeRow 的项目属于同一 `sourceGroup`；
- 一个分组在网格内换行；
- 下一个分组总是从新行开始；
- 可切换为“紧凑排列”，此时所有项目连续填充。

### FR-10 PSD 生成

- 始终创建新文档；
- 文档尺寸按布局预先计算；
- 每项创建一个画板；
- 画板名等于 assetCode；
- 每个画板置入一个嵌入式智能对象；
- 置入后按可见边界等比缩放；
- 移动到目标视觉中心；
- 智能对象放入对应画板；
- 任务支持取消；
- 生成失败须报告具体 assetCode 和步骤。

### FR-11 分卷

模板包含 `maxArtboardsPerDocument`，默认 100。

- 优先在 sourceGroup 边界分卷；
- 单个分组超过上限时允许在组内分卷；
- 每卷重新从坐标 (0,0) 排列；
- 后续分卷自动使用 `_02.psd`、`_03.psd` 等连续编号。

### FR-12 保存输出

生成流程只输出 PSD/PSB，不自动创建同名 Manifest 或 CSV。需要排查问题时由“运行与诊断”模块按需导出诊断包。

## 7. 非功能需求

### NFR-01 可靠性

- 不读取样式；
- 不依赖 Excel 桌面程序；
- 不依赖网络；
- 所有相对路径必须统一规范化；
- 不吞掉单项错误；
- 生成过程不可修改用户原文档。

### NFR-02 性能与内存

- 首次只解析工作簿索引；
- 选定工作表后才解析该 sheet；
- 只为当前页缩略图解压图片；
- 生成时按项顺序解压并写入临时目录；
- 每项置入完成后释放内存引用；
- 默认缩略图分页 50 项；
- 对超过 250 MB 的工作簿显示“大文件模式”提示。

### NFR-03 可诊断性

提供“导出诊断包”：

- parser summary JSON；
- Manifest；
- 错误日志；
- 不包含未选择图片的原始二进制。

### NFR-04 兼容性

- 默认最低 Photoshop 版本暂定 25.0；
- 项目开始前由团队确认实际最低版本；
- Windows 和 macOS 都必须测试；
- 路径与文件名按两平台非法字符规则清洗。

### NFR-05 安全与隐私

- 插件 Manifest 不申请 network 权限；
- 文件只在用户选择的本地文件、插件临时目录和输出目录间流转；
- 临时文件在任务结束或下次启动时清理。

## 8. 非目标

MVP 明确不做：

- 腾讯文档链接直贴；
- 腾讯文档 API 和 OAuth；
- 自动回写排期状态；
- AI 语义识别；
- 自动绘图；
- 资产相似搜索；
- 自动描边和矢量化；
- 更新已有 PSD；
- 任意 Excel 模板识别；
- 浏览器插件；
- Web 管理后台。

## 9. 验收场景摘要

1. 导入两个现有大工作簿，能列出全部工作表；
2. 在越南工作表识别 `ds_vietnam1`、`ds_vietnam3`、`ds_vietnam52` 并关联正确图片；
3. 选择与 `02.psd` 相同的 43 项，生成 43 个正确命名画板；
4. 保留两个源分组，文档尺寸为 2380 × 1140；
5. 每画板一个嵌入式智能对象；
6. 遇到多个图片候选时，上排标记为参考图、下排标记为项目图并默认选择项目图；
7. 重复 assetCode 阻断生成；
8. 用户取消时停止并给出已完成数量；
9. 输出目录只产生计划内的 PSD 分卷，不附带 Manifest 或 CSV。
