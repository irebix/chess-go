# 标准网格画布｜Photoshop 真机验收清单

自动化测试不能替代 Photoshop DOM、置入智能对象、图层层级和 PSD 重开行为的宿主验收。使用 `dist/` 当前构建，依次核对：

## 1. 初始化与持久化

- [ ] 新建或打开一份无画板、尺寸严格为 `1780 × 1188 px` 的普通 PSD。
- [ ] 面板显示“检测到标准尺寸画布，尚未初始化为棋子go标准网格”。
- [ ] 点击“初始化标准网格”，确认只新增隐藏顶层组“棋子go｜标准网格画布数据”和隐藏文本层“棋子go｜网格配置（请勿删除）”。
- [ ] 确认现有底板和设计图层没有移动、缩放、改名或改变可见性。
- [ ] 保存、关闭并重新打开 PSD，确认仍识别为 `12 × 8` 标准网格。

## 2. 普通手工内容与辅助层

- [ ] 保留一个名称类似“图层 3 拷贝 4”、边界覆盖完整 `1780 × 1188 px` 画布的网格底板，点击“刷新布局”，确认底板不占用任何格。
- [ ] 手工粘贴一个顶层像素层到 `R01C01`，点击“刷新布局”，确认该格计为占用。
- [ ] 隐藏该设计层后再次刷新，确认仍占用。
- [ ] 将旧方案移入顶层 `__ARCHIVE__`，刷新后确认不再占用。
- [ ] 分别建立 `__GRID__`、`__BACKGROUND__`、`__NOTES__`、`__REFERENCES__` 顶层辅助内容，确认均不参与占用。

## 3. 多格内容与间距

- [ ] 放置一个主体位于单格、只轻微越过格线的棋子，刷新后确认只占主体所在格。
- [ ] 放置一个横跨 2 × 2 格子实体区域的大对象，刷新后确认四格均占用。
- [ ] 放置只位于 4 px 间距中的小对象，确认相邻实体格不会被误判。

## 4. AI初稿单个插入与行绑定

- [ ] 在已初始化标准网格 PSD 中准备一条至少 4 个棋子的链，并取得一列 Holopix/Flux 或 GPT Image 2 候选。
- [ ] 点击链中第 3 个棋子的候选，确认显示第一条 12 格全部无内容的建议空行，以及“插入整排 / 插入当前”两个按钮。
- [ ] 点击“插入当前”，确认只把第 3 个棋子放入该行 `C03`，`C01–C02` 仍为空且没有把棋子前移。
- [ ] 确认 Photoshop 新建一个以链名称开头的顶层 `AI初稿` 图层组，第 3 个棋子的智能对象位于组内；再次更新该链时继续使用原组，不产生第二个组。
- [ ] 确认结果为嵌入式智能对象、完整位于 `144 × 144 px` 实体区域、保持比例、不裁切、中心对齐。
- [ ] 使用小于 144 px 的源图，确认默认不放大。
- [ ] 再点击同一条链的任意候选，确认仍显示刚才绑定的同一排，按钮变为“更新整排 / 更新当前”，不会建议第二排。
- [ ] 插件热重载或保存、关闭并重开 PSD 后再次选择同一条链，确认仍能从智能对象图层标记找回绑定排。

## 5. 写入前临时变更

- [ ] 首次选中候选并取得建议空行后，在 Photoshop 手工向该行任一格加入内容。
- [ ] 再点击“插入当前”或“插入整排”，确认插入取消、手工内容未被覆盖，并显示新的完整空行或无空行提示。
- [ ] 链已绑定后，在尚未写入的固定链位手工加入其他内容；再尝试“更新当前”或“更新整排”，确认停止覆盖且不会为该链另开第二排。

## 6. AI初稿整排插入与替换

- [ ] 确认一级折叠栏显示“AI初稿”，不再显示“AI 生成”。
- [ ] 同时打开来源画板 PSD 和标准网格 PSD，在标准网格标签页保持活动时重载插件，确认 `AI初稿` 入口仍显示，并自动恢复旁边已打开来源 PSD 的物品链。
- [ ] 暂时没有可识别的来源画板 PSD 时，确认 `AI初稿` 入口仍显示，内容区提示保持来源画板 PSD 打开或先切换到来源 PSD，而不是整栏消失。
- [ ] 保持来源画板 PSD 打开并切换到标准网格 PSD，确认“继续生成未完成”按钮、候选数量步进器和 Flux 单格重生成仍可用；生成期间活动文档保持标准网格，候选进度正常增加。
- [ ] 在标准网格仍活动时关闭来源画板 PSD，再尝试继续生成，确认提交前停止并提示来源 PSD 已关闭或节点结构变化，不发生付费提交。
- [ ] 分别使用 Flux 与 GPT Image 2，准备一条 8 个棋子的链，并让同一候选列的第 2 个棋子缺少图片、第 3 个棋子有图片。
- [ ] 点击该列任意可用候选，确认只建议一条 12 格全部无内容的完整空行，链位置范围为该行 `C01–C08`。
- [ ] 点击“插入整排”，确认第 1 个进入 `C01`、`C02` 保持空白、第 3 个进入 `C03`，后续棋子均不向前补位。
- [ ] 确认本次整排的全部智能对象都进入同一个顶层 `AI初稿` 图层组，图层组名称包含当前链名称与绑定行标记。
- [ ] 为同一链生成或选择另一候选列，确认按钮变为“更新整排 / 更新当前”；点击“更新当前”只替换本次点击链位的智能对象内容。
- [ ] 点击“更新整排”，确认有本列候选的已有链位被替换、空链位被按固定列补齐；本列仍缺失候选的位置保持原状。
- [ ] 让每一行至少有一个已有内容，确认即使存在很长的连续空位也拒绝规划，并提示没有完整空行。
- [ ] 在首次建议行的 `C09–C12` 或任一链目标格预先加入内容，确认整批开始前被阻止，不发生写入，并重新寻找其他完整空行。
- [ ] 回到画板 PSD，确认点击候选仍只按原画板节点回填，不套用整行规则。
- [ ] 模拟批量中的单张置入或替换失败，确认整个“AI初稿”历史事务回滚，不留下半排新旧混合结果。

## 7. AI编辑

- [ ] 在画板 PSD 和标准网格 PSD 各执行一次 AI编辑。
- [ ] 确认写入前重新验证来源文档和来源图层，结果与保存的来源边界 contain/居中，原图保留。
- [ ] 确认结果没有寻找空格，网格空闲格计数不因编辑版本增加。
- [ ] 删除来源或切换文档后，确认结果保留为就绪图片且不会写入错误文档。

## 8. AI勾线

- [ ] 在标准网格 PSD 生成描边，确认 Shape 位于来源图层正上方，路径几何仍与来源重合。
- [ ] 确认没有寻找新格，也没有改变网格占用缓存。
- [ ] 删除来源或切换文档后，确认描边保留为就绪结果且不会写入错误文档。

## 9. AI细化

- [ ] 在标准网格中分别选择完全位于格内、轻微越格和明显大于单格的来源执行 AI细化。
- [ ] 确认结果仍尽量保持来源尺寸与中心；轻微越格时只做最小平移，大于单格时等比缩入来源主格，最终四边均不越出该格实体区域。
- [ ] 对包含多个来源图层的组执行细化，确认每张结果分别按自身来源所在主格约束，不把整组误算为同一格。
- [ ] 回到画板 PSD，确认 AI细化仍按原来源边界对齐，不套用标准网格约束。

## 10. 普通画布边界

- [ ] 打开其他尺寸或未初始化的普通 PSD，确认显示“当前不是棋子go标准网格画布，无法自动定位”。
- [ ] 点击现有候选、插入 AI编辑结果或插入就绪描边，确认 Photoshop 不新增内容且 AI 结果保留。
- [ ] 修改一份已初始化 PSD 的画布尺寸，确认提示“网格配置与当前画布尺寸不一致，已停止自动定位”。

## 11. 画板回归与性能

- [ ] 回到现有棋子归档画板 PSD，确认候选点击仍调用原画板回填，创建缺失智能对象、替换和几何回滚行为不变。
- [ ] 普通绘制、移动、粘贴和变换期间观察日志，确认没有标准网格完整布局扫描。
- [ ] 只有首次进入、准备放置、缺少缓存或点击“刷新布局”时出现 `chess-go.grid.layout-scan`；正式放置前只出现目标格 `target-recheck`。
